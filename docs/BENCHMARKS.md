# Benchmarks

Latency numbers for each phase of the ranker, generated by
`scripts/bench.py`. Regenerate after any change that could affect
hot-path performance — feature additions, embedding model swaps, or
LightGBM hyperparameter changes — and paste the new table below.

## How to regenerate

```bash
pip install -e ".[ml]"
python scripts/bench.py --sizes 10,100,500,1000 --repeats 50 \
    --with-embeddings \
    --learned-model models/learned_ranker.joblib > /tmp/bench.txt
```

Then paste `/tmp/bench.txt` into the section below, replacing the
existing snapshot.

## Notes

- Embedding **precompute** is reported separately because in production
  the `{user_id: vector}` map would live in a vector store and would not
  be paid per request. Per-request latency for `rules+embeddings`
  assumes the cache is already populated.
- The seed CSV has ~1,000 profiles. Larger pools should be benchmarked
  on representative production data once that exists.
- Numbers are wall-clock from `time.perf_counter`, single-threaded,
  CPython 3.11 on the machine that ran the benchmark. Production
  numbers will differ — these are a relative comparison, not an SLA.
